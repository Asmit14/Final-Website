1.What is the Hypertext Transfer Protocol (HTTP)?
Ans: Http is a protocol which allows the fetching of resources, such as HTML documents. It is the fundación of any data exchange on the web, and it is a client-server protocol, which means requests are initiated by the recipient, usually the web browser.

2.Why is HTTP called a stateless protocol?
Ans: Http is called as a stateless protocol because each request is executed independently without any knowledge of the requests that were executed before it, which means once the transaction ends the conection between the browser and the server is lost.

3.Describe the three phases of establishing a session using client-server protocols such as HTTP
Ans: Three phases of establishing a session using client-server protocols such as http are as follows:
a. Establishing a connection In client-server protocols, it is the client which establishes the connection. Opening a connection in HTTP means initiating a connection in the underlying transport layer, usually this is TCP.
b. Sending a client request Once the connection is established, the user-agent can send the request. A client request consists of text directives, separated by CRLF which is divided into three blocks.
c. Structure of a server response After the connected agent has sent its request, the web server processes it and ultimately returns a response. Similar to a client request, a server response is formed of text directives, separated by CRLF, though divided into three blocks: The first line, the status line, consists of an acknowledgment of the HTTP version used, followed by a status request then Subsequent lines represent specific HTTP headers, giving the client information about the data sent.Similarly to the block of HTTP headers for a client request, these HTTP headers form a block ending with an empty line. The final block is a data block, which contains the optional data.

5.What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?
Ans: Http response status codes indicate whether a specific http requests has been successfully completed. They are grouped into five classes:
a. Informational responses
b. Successful responses
c. Redirects
d. Client errores
e. Server errors
The status code 200 indicates that the requests was received and understood and is being processed.
The status code 404 is a http standard response code to indicate that the browser was able to communicate with a given server, but the server could not find what was requested

10.What are Cross-site HTTP requests (CORS) and what requests use CORS?
Ans: Cross Origin Resource Sharing (CORS) is a W3C standard that allows a server to relax the same-origin policy. It is not a security feature, CORS relaxes security. The CORS mechanism supports secure cross-origin requests and data transfers between browsers and servers. Modern browsers use CORS in APIs such as XMLHttpRequest or Fetch to mitigate the risks of cross-origin HTTP requests.
